# Round 15: TUI Refresh

目标：验证新的终端信息层级是否能清楚区分用户输入、系统路由、执行进度、等待状态和结果块。

## 真实验收

1. 用真实 `codex-cli` 运行：
   `./metaclaw.sh start`
2. 输入真实任务：
   `请帮我总结一下当前这个项目的定位`
3. 预期结果：
   用户输入以 `> ` 独立成块显示
4. 预期结果：
   运行摘要显示为一行数字统计：
   `当前执行 1 | 待执行 0 | 已挂起 0 | 阻塞 0`
5. 预期结果：
   输入区上方出现 `status: running codex-cli`
6. 预期结果：
   执行准备与进度使用分层前缀：
   - `→ 正在回忆任务 ...`
   - `→ 派发给 codex-cli...`
   - `· #task_xxx 已启动 codex-cli 执行器`
   - `· 正在等待执行器返回...`
7. 预期结果：
   完成后以结果块收口：
   - `✓ 任务完成 (xx.xs)`
   - `┌─ 任务结果 ─ ...`
   - 结果正文独立显示，不和执行日志混在一起

## 本轮实测记录

- 已用真实 `codex-cli` 跑通单任务场景
- 已观察到：
  - 新数字状态栏
  - `status: idle / running codex-cli`
  - 准备阶段 `→`
  - 执行步骤 `· #task...`
  - 静默等待提示 `· 正在等待执行器返回...`
  - `✓ 任务完成` 结果块
