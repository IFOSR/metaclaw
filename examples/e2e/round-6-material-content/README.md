# Round 6: Material Content Injection

目标：让已 attach 的文本材料不只是“路径存在”，而是能真正进入执行器上下文，影响结果生成。

本轮验收重点：

- 文本材料的关键内容被注入执行器上下文
- 执行器 prompt 同时包含材料路径和材料摘录
- 不可读材料不会导致报错，只保留路径
- 用户能在真实任务里看到材料确实影响了输出

## 场景清单

### 脚本化场景

- `scripts/00-material-content-smoke.txt`
  - 验证文本材料 attach 后，执行器可以基于材料产出更具体结论

### 手动场景

- `manual/01-text-materials-influence-result.md`
  - 验证同一任务在补文本材料前后，输出具体度明显不同

## 本轮通过标准

- 文本材料摘录出现在执行器上下文中
- 执行器回答能引用材料中的事实，而不是只说“没有材料”
- 不支持解析的文件仍可正常 attach，不影响任务流
