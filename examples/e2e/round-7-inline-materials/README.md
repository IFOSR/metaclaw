# Round 7: Inline Materials On Task Creation

目标：让用户在一条自然语言里同时“创建任务 + 指定本地材料路径”，无需先建任务再 `/attach`。

本轮验收重点：

- 自然语言中的本地文件路径会被识别并挂到任务材料
- 新建任务时会明确提示已自动关联多少份材料
- 任务标题/目标仍然可读，不是被路径污染的字符串
- 现有材料注入能力自动生效

## 场景清单

### 脚本化场景

- `scripts/00-inline-materials-smoke.txt`
  - 验证一条输入里同时创建任务并附加两份材料

### 手动场景

- `manual/01-create-task-with-inline-files.md`
  - 验证用户粘贴本地路径时无需额外 `/attach`

## 本轮通过标准

- 输入中的有效本地文件路径被识别为材料
- 新任务详情中能看到这些材料
- 任务说明仍然保留可读语义
