# Round 8: Web Link Materials

目标：让用户在自然语言里直接带网页链接创建任务，链接会像材料一样进入任务视图和执行上下文。

本轮验收重点：

- `https://...` 链接会被自动识别为材料
- 本地文件和网页链接可以在一条输入里同时出现
- 任务标题仍然保持可读
- 任务详情里能看到链接材料

## 场景清单

### 脚本化场景

- `scripts/00-inline-web-links-smoke.txt`
  - 验证创建任务时同时带本地文件和网页链接

### 手动场景

- `manual/01-create-task-with-inline-links.md`
  - 验证无需 `/attach` 也能把网页链接挂到任务材料中

## 本轮通过标准

- 链接被识别并保留原始 URL
- 任务详情中的 `关联材料` 包含 URL
- 文件材料能力不回退
