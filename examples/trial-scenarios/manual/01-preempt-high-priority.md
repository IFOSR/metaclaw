# Manual Scenario 01: High-Priority Preemption

目标：验证单执行器模式下，用户仍可继续输入新任务；高优任务会抢占当前运行任务，原任务进入 `parked`。

## 步骤

1. 启动 Metaclaw：

```bash
./metaclaw.sh start
```

2. 输入一个真实研究任务：

```text
我要先调研下比亚迪与宁德时代在新能源电池层面的市场份额，并输出一个简短结论
```

3. 在它还在执行时，立刻插入一个高优任务：

```text
紧急：帮我分析佛塑科技和新能源电池产业链的关系，先给我一个一页结论
```

4. 执行：

```text
/tasks
```

## 预期

- 输入框在第一个任务运行时仍然可输入
- 第二条高优请求会触发抢占，而不是报“系统忙”
- `/tasks` 中应至少看到：
  - 一个 `当前执行` 任务
  - 一个 `已挂起` 任务
- 抢占提示中会出现：
  - 被抢占的任务 ID
  - 抢占原因

## 建议观察点

- 抢占后的输出文案是否清楚
- 原任务是否进入 `parked`
- 高优任务完成后，是否仍能手动恢复原任务
